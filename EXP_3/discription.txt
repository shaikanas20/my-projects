

1. ADMIN DASHBOARD – TEST CASES & VALIDATIONS

1.1 FIELD: Theme Toggle

Requirement:

The toggle must switch between Light Mode and Dark Mode instantly without page refresh.
Should be accessible via keyboard and mouse click.
When in Dark Mode, all text (including “Welcome, Admin!”) and background elements should adjust to dark theme colors defined in CSS.

Valid Actions:

Clicking the toggle switches to Dark Mode (dashboard, sidebar, header, main text all change).
Clicking again returns to Light Mode.

Invalid Actions:

Toggle does not change colors of some elements – FAIL.
Toggle requires a refresh to take effect – FAIL.

1.2 FIELD: Navigation Links

Requirement:

Links must be clickable and respond visually when hovered.
They must retain readability in both Light and Dark themes.

Valid Actions:

Hover changes link style while keeping contrast.
Clicking opens respective page or section.

Invalid Actions:

Hover color blends with background (low contrast).
Links unreadable in either theme.

1.3 FIELD: Welcome Message

Requirement:

Text “Welcome, Admin!” should be visible and have correct contrast in both themes.
Font size should remain consistent.

Valid Actions:

Dark Mode makes text light-colored, Light Mode keeps it dark.

Invalid Actions:

Text remains dark in Dark Mode (poor contrast).

1.4 FORM / DASHBOARD INTERACTION

Requirement:

Layout must remain intact on all screen sizes.
Theme toggle and navigation must work regardless of device width.

Valid Actions:

Dashboard adjusts well on mobile without overlapping elements.

Invalid Actions:

Elements overlap or text is cut off on small screens.



2. CSS Styling Breakdown: A Structured Overview

2.1 ELEMENT: body (Page Background)

Purpose: Sets base background for the dashboard.

Light Mode: background-color: #f5f5f5; for a clean, neutral tone.
Dark Mode: background-color: #222; for reduced eye strain.

2.2 ELEMENT: .dashboard (Main Layout Container)

Purpose: Wraps all sections (header, sidebar, main content, footer).
Effect: Uses grid or flex layout for sidebar and content separation.

2.3 ELEMENT: .header (Top Bar)

Purpose: Displays title and theme toggle.
Light Mode: Dark text on light background.
Dark Mode: Light text on dark background.

2.4 ELEMENT: .sidebar (Navigation Menu)

Purpose: Holds navigation links.
Light Mode: White background with dark text.
Dark Mode: Dark background with light text.

2.5 ELEMENT: .main (Main Content Area)

Purpose: Displays welcome text and dashboard info.
Effect: Padding for spacing, responsive text size.

2.6 ELEMENT: .footer
Purpose: Stays at the bottom, visible in both themes.


3. LEARNING OUTCOMES

1. How to create a responsive admin dashboard with Light/Dark mode.
2. How to style layouts consistently using separate HTML and CSS files.
3. How to ensure accessibility and theme contrast for all UI elements.
4. How to run UI test cases for dashboard functionality and visual consistency.
